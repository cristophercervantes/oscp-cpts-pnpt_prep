## Windows Sessions

### What is a Session?

When you interact with a Windows system, everything happens within a **session** — a container that tracks who is logged in and what they're doing. There are two main types: **interactive** and **non-interactive**.

### Interactive Sessions

These are sessions where a **real person** logs in using credentials (username + password).

This can happen in 3 ways:

- **Direct logon** — sitting at the machine and logging in physically
- **runas command** — launching a program as a different user from the command line
- **Remote Desktop (RDP)** — logging in over the network as if you were sitting there

### Non-Interactive Sessions

These sessions have **no human behind them** — no password, no manual login. Windows uses them automatically to run background tasks and services.

There are 3 built-in non-interactive accounts:

|Account|Also Known As|Power Level|
|---|---|---|
|Local System|`NT AUTHORITY\SYSTEM`|Highest — even above local admins|
|Local Service|`NT AUTHORITY\LocalService`|Low — similar to a regular local user|
|Network Service|`NT AUTHORITY\NetworkService`|Low locally, but can authenticate on the network|

#### When are these used?

- Starting **Windows services** at boot
- Running **scheduled tasks** in the background
- Anything the OS needs to do **on its own**, without a user being present

## Interacting with the Windows OS

### 3 Ways to Interact with Windows

### 1. GUI (Graphical User Interface)

The GUI is the visual, point-and-click interface most people are familiar with.

- Introduced in the **late 1970s** by Xerox, later adopted by Apple and Microsoft
- Made computers accessible to everyday users — no commands to memorize
- Admins commonly use it for things like **Active Directory**, **IIS**, and **databases**

### 2. Remote Desktop Protocol (RDP)

RDP lets you connect to another computer **over a network** and use its GUI as if you were sitting right in front of it.

- Microsoft's own proprietary protocol
- Uses **port 3389**
- Common uses: remote administration, working from home (usually over a VPN)

### 3. Command Line

Gives you **more control and flexibility** than the GUI. Two main options:

#### CMD (Command Prompt)

- The classic Windows shell (`cmd.exe`)
- Good for quick one-off commands and basic scripting
- Run `help` to see all commands, or `help <command>` / `<command> /?` for specific usage

```
ipconfig /?        → shows all options for ipconfig
help schtasks      → shows how to manage scheduled tasks
```

#### PowerShell

- A more **powerful, modern shell** built on the .NET Framework
- Designed specifically for system administrators
- Uses **cmdlets** — small built-in commands in `Verb-Noun` format


```powershell
Get-ChildItem              # list current directory
Get-ChildItem -Recurse     # list all subdirectories too
Get-ChildItem -Path C:\Users\Admin\Documents
```

### PowerShell:  Concepts

#### Aliases

Shortcuts for longer cmdlets — many mirror Linux commands so they feel familiar.

|Alias|Full Cmdlet|
|---|---|
|`ls` or `gci`|`Get-ChildItem`|
|`cd` or `sl`|`Set-Location`|
|`cat`|`Get-Content`|

View all aliases with `Get-Alias`, or create your own with `New-Alias`.

#### Getting Help

```powershell
Get-Help <cmdlet-name> -Online    # opens browser docs
Update-Help                        # downloads help files locally
```

#### Running Scripts

```powershell
.\script.ps1                      # run directly
Import-Module .\script.ps1        # load all functions into session
Get-Module                        # see all loaded modules
```

#### Execution Policy

A **security feature** that controls which scripts can run. It is **not** a hard security barrier — it can be bypassed — but it's good to understand.

|Policy|What it allows|
|---|---|
|`Restricted`|Commands only, no scripts|
|`RemoteSigned`|Local scripts run freely; downloaded ones need a signature|
|`Bypass`|Everything runs, no warnings|
|`AllSigned`|All scripts must be signed by a trusted publisher|

Check current policy:

```powershell
Get-ExecutionPolicy -List
```

Temporarily bypass for your session only:

```powershell
Set-ExecutionPolicy Bypass -Scope Process
```

> `-Scope Process` means the change only lasts for your current session — it resets when you close PowerShell. Nothing is permanently modified.


## Windows Management Instrumentation (WMI)

### What is WMI?

WMI is a built-in Windows subsystem that gives administrators a powerful way to **monitor, manage, and control** systems — both locally and remotely across a network. It has come pre-installed on Windows since **Windows 2000**.

Think of it as a behind-the-scenes engine that collects information about everything on a system and lets you act on it.

### What Can You Do With WMI?

- Check status of local or remote systems
- Configure security settings remotely
- Manage user and group permissions
- Change system properties
- Execute code / run processes
- Schedule tasks
- Set up logging

### How WMI is Built — Key Components

|Component|What it Does|
|---|---|
|**WMI Service**|Runs at boot; acts as the middleman between everything|
|**WMI Repository**|The database storing all static WMI data|
|**WMI Providers**|Watch for events/data on specific objects|
|**Classes**|Used by providers to pass data to the WMI service|
|**Methods**|Actions attached to classes (e.g. start/stop a process)|
|**CIM Object Manager**|Fetches data from providers and hands it to the requester|
|**WMI API**|Lets applications tap into the WMI infrastructure|
|**WMI Consumer**|Sends the actual queries through the CIM Object Manager|

### Two Ways to Use WMI

#### 1. WMIC (Command Line)

`WMIC` is the command-line interface for WMI. Note: it is **deprecated** in newer Windows versions, but still works.


```cmd
wmic /?                          # list all commands and switches
wmic computersystem get name     # get the hostname
wmic os list brief               # brief overview of the OS
```

Example output:

```
BuildNumber  Version     SystemDirectory
19041        10.0.19041  C:\Windows\system32
```

#### 2. PowerShell (Get-WmiObject)

PowerShell gives you cleaner, more flexible access to WMI.


```powershell
# Get OS information
Get-WmiObject -Class Win32_OperatingSystem | select SystemDirectory,BuildNumber,Version | ft
```


```powershell
# Rename a file using WMI methods
Invoke-WmiMethod -Path "CIM_DataFile.Name='C:\users\public\spns.csv'" -Name Rename -ArgumentList "C:\Users\Public\new_name.csv"
```

A `ReturnValue` of **0** means the command succeeded.

### WMIC Syntax Pattern

WMIC follows a simple pattern:

```
wmic <alias> <verb> <adverb>
```

- **Alias** — the object you're targeting (e.g. `os`, `computersystem`)
- **Verb** — what to do (e.g. `list`, `get`)
- **Adverb** — how much detail (e.g. `brief`, `full`)


>WMI is most powerful for **remote management** — you can query or control another machine on the network without ever touching it physically. This is why it's heavily used by both system admins and, offensively, by attackers doing reconnaissance.
