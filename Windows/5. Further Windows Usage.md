## Microsoft Management Console (MMC)

### What is MMC?

MMC is a built-in Windows tool that acts as a **central hub for administrative tools**. Instead of opening ten different utilities separately, MMC lets you combine the ones you need into a single, customized console.

It has been around since **Windows Server 2000** and runs on all modern Windows versions.

### The Core Concept — Snap-ins

MMC works around **snap-ins** — individual administrative tools that you plug into the console. You pick only what you need, arrange them how you want, and MMC becomes your personalized management dashboard.

Snap-ins can manage both **local and remote systems**.

### How to Use MMC

**Open it:** Type `mmc` in the Start menu → opens blank by default

**Add snap-ins:**

```
File → Add or Remove Snap-ins
```

When adding each snap-in, Windows will ask:

- Manage the **local computer**, or
- Manage a **remote computer** on the network

**Save your setup:**

- Once configured, save as a `.msc` file
- Saved by default to **Windows Administrative Tools** in the Start menu
- Next time you open MMC, just load your saved `.msc` — everything reloads exactly as you left it

|Without MMC|With MMC|
|---|---|
|Open each tool separately|Everything in one window|
|Rebuild your workspace every time|Save and reload your layout as a `.msc` file|
|Manage one system at a time per tool|Mix local and remote management in one console|
|Tools scattered across the system|Custom console distributed to other users/admins|
## Windows Subsystem for Linux (WSL)

### What is WSL?

WSL is a Windows feature that lets you run a **real Linux environment directly inside Windows** — no virtual machine, no dual boot required. It was built for developers who needed Linux tools while staying on a Windows machine.

Available on **Windows 10** and **Windows Server 2019+**.

### Two Versions

- **WSL 1** — Original version, translated Linux calls to Windows
- **WSL 2** _(released May 2019)_ — Ships with a **real Linux kernel** using Hyper-V, making it significantly more compatible and faster

### Installing WSL

Run this in PowerShell **as Administrator:**

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
```

Then get a Linux distro either by:

- Downloading from the **Microsoft Store** (easiest), or
- Manually downloading and unpacking from the command line


### Using WSL

Once installed, just type `bash` in any Windows console to drop into a Linux shell:
```bash
# You get a full Linux directory structure
ls /
# bin  boot  dev  etc  home  lib  mnt  opt  root  sbin  srv  tmp  usr  var
```


```bash
# Check your Linux kernel version
uname -a
# Linux WS01 4.4.0-18362-Microsoft ...
```

It looks, feels, and behaves like a real Linux system — because it essentially is one.

### Accessing Windows Files from WSL

Your Windows drives are mounted under `/mnt`, so crossing between the two worlds is seamless:


```bash
cd /mnt/c        # this is your Windows C: drive
ls               # browse your Windows files from Linux
```

From inside WSL you can also run Linux package managers, install tools, run scripts — anything you'd do on a normal Linux machine.
