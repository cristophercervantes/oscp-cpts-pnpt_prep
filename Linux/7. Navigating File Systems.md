# Introduction
## 1. Linux vs. Windows Directory Structure

The way Linux organizes data is fundamentally different from Windows.

| **Feature**     | **Windows**                     | **Linux**                                     |
| --------------- | ------------------------------- | --------------------------------------------- |
| **Top Level**   | "This PC" / "My Computer"       | The **Root Directory**, symbolized by **`/`** |
| **Drive Logic** | Assigned letters (C:, D:, etc.) | Physical devices are "mounted" as directories |
| **User Data**   | `C:\Users\`                     | `/home/`                                      |

## 2. Key Directories to Know

- **`/` (Root):** The starting point of the entire filesystem. Every other directory is a "child" of root.
    
- **`/boot`:** Contains files required to start the computer.
    
- **`/home`:** Contains personal storage for all regular users.
    
- **`/root`:** The home directory for the administrator (superuser).
    

## 3. The Home Directory (`~`)

Every user is assigned a specific folder under `/home` (e.g., `/home/sysadmin`).

- **Purpose:** This is where you have full control to create, edit, and delete files.
    
- **Symbol:** The tilde **`~`** character is a shortcut for "my home directory."
    
- **Automatic Placement:** When you open a terminal, you are usually placed in your home directory by default.
    
- **Referring to others:** `~bob` is a shortcut for `/home/bob`.
    

## 4. Navigation Commands

To move through the hierarchical structure, use these three essential commands:

### `pwd` (Print Working Directory)

Shows you exactly where you are currently located in the filesystem.

> **Example:** If you see `/home/sysadmin`, you are in the sysadmin's home folder.

### `cd` (Change Directory)

Used to move between folders.

- **Syntax:** `cd [path]`
    
- **Specific Folder:** `cd Documents` moves you into the Documents folder.
    
- **Go Home:** Simply typing **`cd`** with no arguments returns you to your home directory immediately.
    
- **Error Handling:** If you try to enter a folder that doesn't exist, you will see: `-bash: cd: [Name]: No such file or directory`.
    

### `ls` (List)

Shows the contents of a directory.

- **Example:** `ls /` displays all folders located at the root level.
    

## 5. Understanding the Command Prompt

The terminal prompt often gives you clues about your location:

- `sysadmin@localhost:~$` — The **`~`** indicates you are in your **home directory**.
    
- `sysadmin@localhost:~/Documents$` — Indicates you are in the **Documents** folder inside your home.

# Paths
## 1. Absolute vs. Relative Paths

The most important distinction in Linux navigation is where your "directions" begin.

### Absolute Paths

- **Definition:** The full address of a file or directory starting from the very top.
    
- **Starting Point:** Always starts with the **root slash `/`**.
    
- **Pros:** It works no matter which directory you are currently in.
    
- **Example:** `/home/sysadmin/Documents/School/Art`
    

### Relative Paths

- **Definition:** Directions based on your **current location**.
    
- **Starting Point:** Never starts with a `/`. It starts with a folder name or a shortcut.
    
- **Pros:** Much faster to type if you are already "near" your destination.
    
- **Example:** If you are in `Documents`, just type `cd School/Art`.
    

## 2. Path Shortcuts

Linux provides two special "hidden" directory references that exist in every single folder on the system:

|**Shortcut**|**Representation**|**Best Use Case**|
|---|---|---|
|**`..`** (Double Dot)|**The Parent Directory**|Moving "up" one level towards the root.|
|**`.`** (Single Dot)|**The Current Directory**|Used when a command requires you to specify "right here."|

### Using the Parent Shortcut (`..`)

You can chain these together to move up multiple levels at once:

- `cd ..` — Move up 1 level.
    
- `cd ../..` — Move up 2 levels.
    
- `cd ../../Downloads` — Move up two levels, then go into the "Downloads" folder.

# Listing Files in a Directory
## 1. Sorting by File Size

While `ls` defaults to alphabetical sorting, you can sort by file size using the **`-S`** (capital S) option.

- **Behavior:** Displays the largest files first.
    
- **Best Practice:** Pair with **`-l`** so you can actually see the size metadata you are sorting by.
    
- **Human-Readable:** Add **`-h`** to convert the raw byte count into KB, MB, or GB.
    

> **Example:** `ls -lSh /etc/ssh`
> 
> (Lists files from largest to smallest with easy-to-read sizes like "541K").

## 2. Sorting by Time/Date

To see which files were recently worked on, use the **`-t`** option.

- **Behavior:** Sorts by modification time, newest files first.
    
- **Directory Modification:** Remember that a directory's timestamp only changes when a file is **added or deleted** from it, not when a file _inside_ it is edited.
    
- **Precision:** Use **`--full-time`** to see the exact second and nanosecond a file was modified. This automatically triggers the long listing (`-l`) format.
    

## 3. Reversing Sort Order

The **`-r`** option flips the sorting logic. This is incredibly useful for keeping the most relevant files at the bottom of a long terminal output.

|**Command Combination**|**Resulting Order**|
|---|---|
|**`ls -lS`**|Largest $\rightarrow$ Smallest|
|**`ls -lrS`**|Smallest $\rightarrow$ Largest|
|**`ls -lt`**|Newest $\rightarrow$ Oldest|
|**`ls -lrt`**|Oldest $\rightarrow$ Newest (Newest files appear at the bottom)|
