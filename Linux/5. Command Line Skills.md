# Command Line Intro
### 1. The Shell (Command Line Interpreter)

The **shell** acts as the middleman between you and the Operating System. It translates your typed commands into actions the OS can perform.

- **Primary Shell:** **Bash** (Bourne Again SHell) is the most common Linux shell.
    
- **Bash Features:**
    
    - **Scripting:** Running a file containing multiple commands (supports logic like loops/functions).
        
    - **Aliases:** Creating short "nicknames" for long, complex commands.
        
    - **Variables:** Storing data to change how the shell or commands behave.
        
    - **History/Editing:** Recalling and modifying previous commands.
        

### 2. The Command Prompt

The prompt indicates the system is ready for input and provides context:

> **Example:** `sysadmin@localhost:~$`

- **User Name:** `sysadmin`
    
- **System Name:** `localhost`
    
- **Current Directory:** `~` (Tilde is shorthand for the **Home Directory**, e.g., `/home/sysadmin`).
    
### 3. Command Syntax

Most Linux commands follow this standard structure:

`command [options] [arguments]`

- **Command:** The program you want to run (e.g., `ls`).
    
- **Options:** Modify **how** a command works (usually starts with `-` or `--`).
    
- **Arguments:** Tell the command **what** to act upon (like a file or directory name).
    
-  Linux is **case-sensitive**. `LS` is not the same as `ls`.
    

#### The `ls` Command (List)

|**Command Variation**|**Description**|
|---|---|
|`ls`|Lists contents of the current directory.|
|`ls /etc/ppp`|Lists contents of a specific directory (Argument).|
|`ls -l`|**Long listing** (shows permissions, size, etc.).|
|`ls -r`|**Reverse** alphabetical order.|
|`ls -lh`|**Human-readable** sizes (K, M, G) instead of bytes.|

> Options can be combined. `ls -l -r` is the same as `ls -lr` or `ls -rl`.

### 4. Command History & Shortcuts

Bash remembers every command you type in a "history list."

- **Navigation:** Use **Up Arrow (↑)** to scroll through previous commands and **Left/Right Arrows** to edit them.
    
- **The `history` Command:**
    
    - `history`: Shows the full list of previous commands with numbers.
        
    - `history 5`: Shows only the last 5 commands.
        

#### Execution Shortcut:

|**Shortcut**|**Action**|
|---|---|
|`!!`|Re-run the **very last** command.|
|`!n`|Re-run a specific command by its number `n` from the history list.|
|`!-n`|Re-run the command that was `n` steps ago.|
|`!word`|Re-run the **most recent** command that started with "word" (e.g., `!ls`).|

# Variables
### 1. What are Variables?

Variables are temporary "storage containers" in the system memory. They hold data that either provides system information or changes how the shell behaves.

### 2. Types of Variables

|**Feature**|**Local Variables**|**Environment (Global) Variables**|
|---|---|---|
|**Scope**|Only the current shell session.|System-wide; available to all sub-shells/tasks.|
|**Persistence**|Lost when the terminal is closed.|Automatically recreated for new shells.|
|**Convention**|Usually lowercase (e.g., `myvar`).|Usually uppercase (e.g., `PATH`, `HOME`).|
|**Example**|`variable1='Something'`|`HISTSIZE=1000`|

### 3. Key Commands for Variables

- **Assignment:** `variable=value` (Note: **No spaces** around the `=` sign).
    
- **Displaying:** Use `echo $variable_name`. The **`$`** is required to see the _value_.
    
- **Listing Environment Vars:** Use the `env` command.
    
- **Making Global:** Use `export variable_name` to turn a local variable into an environment variable.
    
- **Deleting:** Use `unset variable_name` to remove a variable.
    

### 4. The `PATH` Variable 

The `PATH` variable is a critical environment variable. It acts like a "Search Map" for the shell.

- **Function:** It lists directories where the shell looks for programs/commands.
    
- **The Error:** If you type a command and see `-bash: command not found`, it means the program isn't in any directory listed in your `PATH`.
    
- **Structure:** Directories are separated by a **colon (`:`)**.
    
    - _Example:_ `/usr/bin:/bin:/usr/games`
        
#### Modifying the PATH

If you want to add a new folder (e.g., `/custom/folder`) to your search path without breaking the old ones, you must **append** the existing path:

`PATH=/custom/folder:$PATH`

> **Always include `$PATH` at the end of your new assignment. If you forget it (e.g., `PATH=/custom/folder`), you will lose access to all standard commands like `ls` and `cd` for that session!


# Command Types
The shell doesn't treat every command the same way. When you type a command, the shell looks through a specific hierarchy to find the executable code.

### 1. The `type` Command

To figure out what a command is and where it comes from, use `type [command]`.

- **Purpose:** Identifies if a command is a built-in, an external file, or an alias.
    
- **Tip:** Use `type -a [command]` to see **all** versions of a command (e.g., if a command is both a built-in and an external file).
    
### 2. The Four Categories of Commands

#### A. Internal (Built-in) Commands

These are part of the shell itself. They don't exist as separate files because the shell already knows how to run them.

- **Efficiency:** Very fast because no new process is started.
    
- **Example:** `cd`, `echo` (usually), `exit`.
    
- **`type` output:** `cd is a shell builtin`
    

#### B. External Commands

These are separate binary files stored in the computer's folders.

- **Search Method:** The shell looks for these in the directories listed in your `$PATH`.
    
- **The `which` Command:** Used specifically to find the file path of an external command.
    
    - _Example:_ `which ls` might return `/bin/ls`.
        
- **Manual Execution:** You can run these by typing their full path (e.g., `/bin/ls`) even if they aren't in your PATH.
    

#### C. Aliases

Short "nicknames" for longer commands.

- **Purpose:** To save time on frequently used or complex commands.
    
- **Management:**
    
    - `alias`: View all currently active aliases.
        
    - `alias name='command'`: Create a new temporary alias.
        
- **Persistence:** Aliases created in the terminal disappear when the session ends.
    
- **Example:** `alias ll='ls -alF'`
    

#### D. Functions

Small snippets of code (usually in scripts) that group multiple commands together.

- **Syntax:**
    ```
    function_name () {
       command1
       command2
    }
    ```
    
- **Usage:** Once defined, just type `function_name` to run the whole block.
    
- **Visual Cue:** When typing a function in the terminal, the prompt changes to `>` to show it is waiting for you to finish the block with a `}`.
    

# Quotations
The shell treats certain characters (like `*`, `$`, and ) as "special." Quoting allows you to tell the shell to either ignore that special meaning or handle it differently.

### 1. Double Quotes (`" "`)

Known as **"Weak Quoting."** They protect most special characters from being interpreted by the shell, but they still allow a few specific things to work.

- **What they HIDE:** Glob characters/Wildcards (like `*`, `?`, and `[]`).
    
- **What they ALLOW:** * **Variable Substitution:** `$VAR` will still show its value.
    
    - **Command Substitution:** Backticks (see below) still work.
        
- **Best use:** When you want to group words as one argument but still need to use variables.
    
    - _Example:_ `echo "The path is $PATH"` $\rightarrow$ Shows the actual path.
        

### 2. Single Quotes (`' '`)

Known as **"Strong Quoting."** They are literal. The shell ignores **every** special character inside them.

- **What they HIDE:** Everything (Variables, Globs, Backticks, etc.).
    
- **Best use:** When you want the text to appear exactly as typed, especially if it contains many symbols like `$` or `*`.
    
    - _Example:_ `echo 'The car costs $100'` $\rightarrow$ Shows `$100` (Double quotes would have tried to find a variable named `$1`).
        

### 3. The Backslash (`\`)

Known as the **"Escape Character."** It acts like a "single-character quote."

- **Function:** It turns off the special meaning of the **one** character immediately following it.
    
- **Best use:** When you want to mix literal text and variables in the same line without quoting everything.
    
    - _Example:_ `echo The cost is \$1 and the path is $PATH` $\rightarrow$ The `$` in `$1` is treated as text, but `$PATH` remains a variable.
        

### 4. Backquotes/Backticks (`` ` ` ``)

Used for **Command Substitution**. This tells the shell: "Run the command inside these marks first, and put its output right here."

- **Function:** Replaces the command inside the backticks with its result.
    
- **Alternative Syntax:** Modern Linux users often use `$(command)` instead of backticks, as it's easier to read, though both work.
    
    - _Example:_ `echo Today is` `date` `` $\rightarrow$ Result: _Today is Mon Nov 4..._
        

# Control Statement
Control statements allow you to manage the flow of command execution. In Linux, "success" usually means a command finished without errors, while "failure" means something went wrong (like a file not being found).

### 1. The Semicolon ( `;` ) — "Run regardless"

The semicolon is used to run multiple commands sequentially.

- **Logic:** `Command 1` finishes, then `Command 2` starts.
    
- **Result:** It does **not matter** if the first command succeeded or failed; the next one will always run.
    
- **Example:** `cal 1 2030; cal 2 2030`
    
    - _Useful for:_ Running a batch of unrelated tasks in one line.
        

### 2. Double Ampersand ( `&&` ) — "Logical AND"

The `&&` operator makes the second command dependent on the success of the first.

- **Logic:** "Run `Command 2` **only if** `Command 1` was successful."
    
- **Example:** `ls /etc/ppp && echo "Directory found"`
    
    - If the directory exists, you see the files AND the word "success."
        
    - If it doesn't exist, you see an error message, and the `echo` command is skipped.
        
- **Best use:** For safety—e.g., "Change to this directory AND delete everything inside" (you wouldn't want to delete things if the directory change failed!).
    

### 3. Double Pipe ( `||` ) — "Logical OR"

The `||` operator is the opposite of the ampersand.

- **Logic:** "Run `Command 2` **only if** `Command 1` fails."
    
- **Example:** `ls /etc/junk || echo "That folder is missing"`
    
    - If the first command succeeds, the shell skips the second command.
        
    - If the first command fails, the shell runs the second command as a fallback.
        
- **Best use:** Error handling—e.g., "Try to open this file OR print an error message if you can't."

# Some essential Commands
### 1. `whoami`

- **Purpose:** Displays the username of the current user.
    
- **Use Case:** If you switch between different user accounts (like `sysadmin` and `root`), this confirms which account is currently active in the terminal.
    
- **Example:**
    ```
    sysadmin@localhost:~$ whoami
    sysadmin
    ```
    

### 2. `uname` (Unix Name)

This command prints system information. By itself, it is very sparse.

- **`uname`**: Displays the **Kernel Name** (typically `Linux`).
    
- **`uname -n`**: Displays the **Network Node Hostname**. This is the name assigned to the computer on the network.
    
- **`uname --nodename`**: This is the **full-word equivalent** of `-n`. Both do the exact same thing.

### 3. `pwd` (Print Working Directory)

- **Purpose:** Displays the full **absolute path** of the directory you are currently in.
    
- **Why use it:** Since the prompt doesn't always show the full path (often just the name of the current folder), `pwd` helps you find exactly where you are starting from the root (`/`).
    
- **Example:**
    ```
    sysadmin@localhost:~/Documents$ pwd
    /home/sysadmin/Documents
    ```
    

### 4. `cd` (Change Directory)

- **Purpose:** Used to move from one directory to another.
    
- **Usage:** `cd [path]`
    
- **Shortcut Behaviors:**
    
    - **`cd` (by itself):** Returns you instantly to your **Home directory** (`~`).
        
    - **`cd ~`:** Also returns you to your Home directory.
        
    - **`cd ..`:** Moves you up one level (to the "parent" directory).
        
    - **`cd -`:** Moves you back to the **previous** directory you were in (like a "back" button).
