# Operating Systems 
- OS = software that manages hardware + software resources on a computing device
- Enables functional computing system
- Modern OS features:
  - Multi-tasking (schedules programs, shares CPU for apparent simultaneity)
  - Provides standard services (e.g. print jobs, file access)
  - Accepts & executes properly requested functions from users/programs

## Scope & Complexity
- Applies to any device that boots and runs software
- Ranges from: simple set-top box menu → firewall → mobile phone → desktop → server → supercomputer → parallel clusters
- Desktop/server OS more complex than single-purpose embedded OS

## Major Choices for Users (2026)
Three primary desktop/server operating systems:

- Microsoft Windows
  - Proprietary code base
  - Not UNIX- or Linux-based
  - Unique underlying architecture

- Apple macOS
  - Fully-qualified UNIX distribution
  - Based on BSD Unix + Darwin kernel
  - Large proprietary additions by Apple
  - Runs on Apple-optimized hardware only

- Linux
  - Open-source kernel
  - Hundreds of distributions (distros)
  - Customized/optimized for specific tasks (desktop, server, embedded, etc.)
  - Not a single OS — family of packages

## User Interaction
- Everyday tasks similar across all three:
  - Point-and-click GUI for productivity
  - File management, browsing, apps behave comparably
- Differences minor for basic users

## Administration & Command Line
- Windows:
  - Mostly GUI-based admin (Control Panel, Settings, MMC)
  - Some CLI equivalents (PowerShell, cmd)

- macOS & Linux:
  - Terminal-heavy for system admin tasks
  - UNIX-style commands (many shared between macOS/Linux)
  - UNIX-familiar admins can work on Linux (and vice versa) with minimal adjustment

- Cross-compatibility:
  - Many Linux CLI functions have Windows equivalents
  - Efficient admins use typed commands on all platforms

## Role

First decision: purpose of the machine

- **Desktop / Workstation**
    - User sits at console
    - Runs productivity apps, web browsing → GUI-focused, familiar desktop environment
- **Server**
    - Accessed remotely by many users
    - Provides services to remote clients
    - Usually rack-mounted, shared console/keyboard/monitor
    - Console access only for config & troubleshooting
    - Runs primarily CLI → frees resources for serving clients

## Function

Determine what the machine must do

- Specific software or tasks required
- Scale: single machine vs hundreds/thousands identical
- Skill set of managing team / administrators

## Life Cycle & Maintenance

- Release cycle: periodic OS/software upgrades
- Maintenance cycle: vendor support duration for older versions
- Enterprise servers: major upgrades expensive & time-consuming → Often replace hardware instead (performance gain justifies cost)
- Proactive orgs maximize ROI on human + monetary capital
- Virtualization: one physical host → dozens/hundreds VMs → reduces space/power, enables automation (create/configure/deploy/remove VMs via scripts)
- Cloud providers (AWS, Azure, Rackspace): further reduces need for physical upgrades → pay for what you use, automated desktop/server management

## Stability

Software release types

- **Beta**: many new/un-tested features → used by developers needing latest features, testing new deployments
- **Stable**: tested in field, reliable → default for production servers unless specific new feature outweighs risk

Open source

- Early peer review releases
- Quick move to testing/production → fast feedback & fixes

Proprietary

- Often secret development
- Public beta near final release

## Compatibility

Backward compatibility: later OS can run software from earlier versions

- Critical when upgrading OS but cannot upgrade application (cost or unavailable)
- Open source norm: prioritize backward compatibility, break only as last resort
- Library versioning: new releases keep old functions/interfaces → existing software continues to work

## Cost

Always major factor

- Microsoft: annual licensing per user/server/software
- Other proprietary: similar fees
- Influences: hardware availability, staff skills, purchase/maintenance cost, future needs
- Virtualization + outsourced/cloud: pay only for usage → controls costs, leverages external expertise

## Interface

Historical progression

- Early: switches, plugboards
- Then: punch cards
- Then: text-based terminals (like modern Linux CLI)
- GUI: pioneered Xerox PARC 1970s, popularized by Apple 1980s

Today

- Most OS offer both GUI & CLI
- Consumer OS (Windows, macOS): designed to hide CLI from average user
- Server/admin-focused: CLI primary, GUI optional/secondary

# Microsoft Windows

## Desktop Versions

- Current version: Windows 11
- Release pattern: major versions every few years (not twice-yearly like most Linux distros)
- Total versions since 1985: 16
- Backward compatibility: strong priority
    - Bundles virtual machine tech to run older software

## Server Versions

- Current: Windows Server 2019 (named by release year)
- GUI available but optional
- Recent improvements (response to Linux competition):
    - PowerShell: advanced command-line scripting
    - Windows Subsystem for Linux (WSL): run Linux tools natively
- Optional Desktop Experience package: turns server into productivity desktop-like environment
- Enterprise push: integrate with Azure cloud service

# Apple macOS

## Base & Certification

- Partially based on FreeBSD project
- UNIX-certified
- Known for ease of use → favored in schools, small businesses, limited IT support environments

## Strengths & Use Cases

- Robust UNIX underpinnings → popular with programmers
- Creative industries (graphics, video production): application choice often drives OS decision
- Stable platform: tight hardware-software integration + strict app programming standards → fewer compatibility issues for compute-intensive work

## Server Side

- macOS Server: add-on to desktop macOS
- Targeted at smaller organizations
- Features:
    - Collaboration tools
    - Access control for shared resources
    - Integration with iOS devices (iPhone, iPad)

## Enterprise Adoption

- Some large IT departments allow macOS choice
- Users often need less support than Microsoft productivity setups
- Healthy third-party software vendor support due to continued popularity

## Linux Distributions 

What is a Distribution? A Linux distribution (distro) is a complete operating system package that includes:
• The Linux Kernel: The core "engine" that talks to hardware
• Utilities & Management Tools: Software to run the system.
• Package Manager: Tools to install and update software.
## Enthusiast vs. Enterprise
• Enthusiast Distros: Focus on the latest features and fast update cycles (e.g., Fedora, openSUSE Tumbleweed). Usually community-supported.

• Enterprise Distros: Focus on stability, security, and long-term support (5-13 years). Often offer paid commercial support (e.g., Red Hat, SUSE, Ubuntu LTS).
## Stability Level:
• Stable: Software that is well-tested and reliable.
• Testing/Beta: A middle ground for finding bugs before a full release.
• Unstable (e.g., Debian "sid"): Experimental versions with frequent changes; used for development.
## User Interfaces
• GUI (Graphical User Interface): Uses windows, icons, and mice. Great for daily desktop use.
• CLI (Command Line Interface): A text-based "terminal" where you type commands into a shell. Efficient for servers and remote management.

## Major Linux Distributions
1. The Red Hat Family (Corporate Focus)
    • Red Hat Enterprise Linux (RHEL): The industry standard for businesses. It is a paid service focused on extreme stability and long-term support.
   • Fedora: Sponsored by Red Hat. It’s the "testing ground" where the newest software features are added first. Great for users who want the latest tech.
   • CentOS / Scientific Linux: Clones of RHEL. They offer the same stability for free but without the official paid support. Scientific Linux is famously used at CERN.

2. The SUSE Family (Professional & Polished)
   • SUSE Linux Enterprise (SLES): A major competitor to Red Hat, very popular in big business environments (especially in Europe).
   • openSUSE: The free, community version. It's known for being very user-friendly and having great configuration tools.

3. The Debian Family (The Community Favorites)
   • Debian: Built entirely by volunteers. It is very strict about only using "Free Open Source Software" and is known for being rock-solid.
   • Ubuntu: Based on Debian. It is the most popular version for regular users. It offers LTS (Long Term Support) versions that stay updated for 5 years, making it very reliable for work.
   • Linux Mint: Based on Ubuntu. It is designed to be very easy for people moving from Windows to Linux.

4. Android (Mobile Linux)
   • The Difference: While Android uses the Linux kernel (the engine), it doesn't use the standard Linux tools (GNU).
   • Incompatibility: You cannot naturally run desktop Linux apps on Android or vice versa because they are built with different "parts" on top of the engine.

5. Specialized Distributions
   • Raspbian: Optimized for the Raspberry Pi. It is the go-to choice for students, teachers, and people building DIY electronics or robots.
   • Linux From Scratch (LFS): Not a download, but a project. It provides instructions to build your own OS from the ground up to learn exactly how Linux works.

  
## Linux in Embedded Systems & IoT
1. What is an Embedded System?
   - An Embedded System is a computer "hidden" inside a device to perform one specific job. Unlike a PC, it doesn't have a screen or keyboard you interact with directly.
   • Examples: Smart TVs, washing machines, factory sensors, and digital recorders (DVRs).

2. Why is Linux the Top Choice?
   • Customizable: You can remove all the parts you don't need (like a desktop or mouse support) to make it tiny enough for a cheap chip.
   • Low Power: It is designed to run on chips that use very little electricity.
   • Compatible: Linux works on almost any type of computer chip in existence.

3. Prototyping with Raspberry Pi
    Small, cheap computers like the Raspberry Pi make it easy for inventors to build new products quickly. What used to take months for a big team can now be done in days by one person using Linux.

4. The Internet of Things (IoT)
   - This is a giant network of embedded devices talking to each other.
   • Industrial Use: Sensors on solar farms or oil wells report data back to a central office.
   • Smart Tech: These devices can monitor safety, adjust processes in real-time, and work with AI to make systems more efficient.
