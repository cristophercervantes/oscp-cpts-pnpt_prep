## **User Accounts & Security Fundamentals**

Linux uses user accounts to manage system security through a "lock and key" model.

- **Identification:** Every person must log in with a unique account.
    
- **Permissions:** Access is controlled via **File Permissions**, which determine what a User, Group, or "Everyone Else" can do with files and directories.
    
- **Groups:** Users are organized into groups to make sharing data easier.
    
- **Data Storage:** User and group information is stored in system database files (like `/etc/passwd` and `/etc/shadow`).
    
- **Monitoring:** Commands exist to track who is logged in and what they are doing to troubleshoot or find unauthorized access.
    

## **Administrative (Root) Accounts**

The **root** user is the "Superuser" with unlimited privileges.

- **The Danger of Root:** Logging in directly as root is discouraged because any mistake can destroy the system.
    
- **GUI Risk:** Running a graphical interface as root is especially dangerous because many background programs run with full authority, increasing the "attack surface" for security threats.
    
- **Best Practice:** Stay logged in as a standard user and only "elevate" to root privileges when performing a specific task.
    

## **The `su` Command (Switch User)**

The `su` command allows you to start a new shell as a different user.

- **Default Behavior:** If no username is specified, it assumes you want to become the **root** user.
    
- **The Login Shell (`-`):** It is highly recommended to use `su -`, `su -l`, or `su --login`. This ensures the new shell loads the target user's specific environment variables and settings.
    
- **Password:** You must enter the **target user's password** (the password of the account you are switching _to_).
    
- **Command Flow:**
    
    1. `su -`
        
    2. Enter root password.
        
    3. Perform tasks.
        
    4. Type `exit` to return to your normal user account.
        

## **The `sudo` Command (SuperUser Do)**

The `sudo` command is the preferred method for administrative tasks in modern distributions like Ubuntu.

- **Execution:** You prefix a single command with `sudo` (e.g., `sudo head /etc/shadow`).
    
- **Password:** Unlike `su`, you enter **your own password**, not the root password.
    
- **The 5-Minute Rule:** Once you enter your password, `sudo` remembers it for 5 minutes, allowing you to run multiple admin commands without re-typing it.
    
- **Key Advantages:**
    
    - **Logging:** Every `sudo` command is logged with a timestamp, the user's name, and the command used. This creates an "audit trail."
        
    - **Safety:** It prevents "accidental root" syndrome because you only get root power for that one specific command.
        
    - **Security:** You don't have to share the actual root password with multiple administrators.

# Sensitive File
## **The `/etc/passwd` File**

This file is the primary database for user account information. It is world-readable, meaning any user can view it to see who is on the system.

### **The 7 Fields of `/etc/passwd`**

Using the example: `sysadmin:x:1001:1001:System Admin:/home/sysadmin:/bin/bash`

1. **Username:** The name used to log in (e.g., `sysadmin`).
    
2. **Password Placeholder:** Usually an `x`. This indicates the actual encrypted password is stored securely in `/etc/shadow`.
    
3. **User ID (UID):** The unique number the system uses to identify the user. Files are "owned" by this number.
    
4. **Primary Group ID (GID):** The ID of the user's main group.
    
5. **Comment (GECOS):** Extra info like the user's real name or phone number.
    
6. **Home Directory:** The user's "starting" folder (usually `/home/username`). Note: Root’s home is `/root`.
    
7. **Login Shell:** The program that runs when the user logs in (usually `/bin/bash`).
    

## **The `/etc/shadow` File**

This file contains sensitive password and "aging" information. **Only root** can read this file.

### **Password Aging & Security Fields**

Example entry: `sysadmin:$6$hash:16874:5:30:7:60:15050:`

|**Field**|**Description**|
|---|---|
|**Username**|Matches the entry in `/etc/passwd`.|
|**Password**|A "one-way" encrypted hash. `*` or `!` usually means it's a system account you can't log into.|
|**Last Change**|Days since Jan 1, 1970 (Epoch) that the password was last updated.|
|**Minimum**|Minimum days a user must wait before they can change their password again.|
|**Maximum**|How long the password is valid (e.g., 30 days). `99999` means it never expires.|
|**Warn**|How many days before expiry the user starts seeing "Please change password" warnings.|
|**Inactive**|The "grace period." Days after expiry where the user can still login but _must_ change the password.|
|**Expire**|An absolute date (in Epoch days) when the account is disabled (useful for temp workers).|

## **Useful Commands for Account Info**

### **1. `grep`**

Use this to quickly find a specific user in the text files.

- `grep "sysadmin" /etc/passwd`
    

### **2. `getent` (Get Entries)**

The `getent` command is more powerful than `grep` because it looks at local files **and** network directory services (like LDAP or Active Directory) if the system is connected to them.

- **Syntax:** `getent [database] [record]`
    
- **Example:** `getent passwd sysadmin` — retrieves the `passwd` line for the sysadmin user.
    
- **Example:** `getent shadow sysadmin` — retrieves password aging info (requires root).

## **System Accounts**

Not every account on Linux is for a human. System accounts are created for services (like web servers or email agents) to run with limited, specific permissions.

- **UID Ranges:**
    
    - **Root User:** Always UID `0`.
        
    - **System Accounts:** Typically UIDs `1` to `499` (or up to `999` on newer systems).
        
    - **Regular Users:** Typically UIDs `500+` or `1000+`.
        
- **Key Characteristics:**
    
    - **No Login:** The shell field in `/etc/passwd` is usually set to `/usr/sbin/nologin` or `/bin/false` to prevent anyone from logging into them.
        
    - **No Password:** The password field in `/etc/shadow` usually contains an asterisk (`*`), meaning the account is locked for authentication.
        
    - **No Home:** They rarely have a standard home directory since they don't store personal files.
        
- **Safety Tip:** Never delete a system account unless you are 100% sure it isn't needed. Removing one can break critical system services.
    
## **Group Accounts**

Groups are used to manage permissions for multiple users at once. Instead of giving 10 people access to a file individually, you put them in a group and give the group access.

### **Types of Group Membership**

1. **Primary Group:** Defined in `/etc/passwd`. Every user has exactly one.
    
2. **Secondary (Supplemental) Groups:** Defined in `/etc/group`. A user can belong to thousands of secondary groups to gain additional permissions.
    

### **The `/etc/group` File Structure**

Example entry: `mail:x:12:mail,postfix`

- **Group Name:** The human-readable name (e.g., `mail`).
    
- **Password Placeholder:** Usually an `x`. Group passwords exist but are rarely used; if they are, they stay in `/etc/gshadow`.
    
- **GID (Group ID):** The unique numerical ID for the group.
    
- **User List:** A comma-separated list of usernames who belong to this group as a **secondary** membership.
    

## **Managing & Viewing Groups**

Like user accounts, you can use `grep` or `getent` to find group information quickly.

- **Using grep:** `grep "groupname" /etc/group`
    
- **Using getent:** `getent group groupname`
    

> **Note:** It is a standard Linux practice for every user to have a "Private Group" created automatically with the exact same name as their username (e.g., user `bob` belongs to primary group `bob`).

## **Identifying Yourself (`id`)**

The `id` command is the quickest way to see who you are logged in as and what permissions (groups) you have.

- **Usage:** `id [username]` (Leave username blank to see your own info).
    
- **The Components:**
    
    - **uid:** Your User ID and Name.
        
    - **gid:** Your Primary Group ID and Name.
        
    - **groups:** A list of all groups you belong to (Primary + Secondary).
        
- **Common Options:**
    
    - `id -g`: Prints **only** the Primary Group ID.
        
    - `id -G`: Prints **all** Group IDs (Primary and Secondary).
        

## **Monitoring Active Users (`who` and `w`)**

If you need to know who is currently using the system, there are two main commands:

### **1. The `who` Command**

Displays basic info about current sessions.

- **tty (Teletype):** Indicates a **local** command-line login.
    
- **pts (Pseudo-terminal):** Indicates a **remote** login (like SSH) or a terminal window inside a GUI.
    
- **Options:** * `who -b`: Shows when the system last **booted**.
    
    - `who -r`: Shows the current **runlevel**.
        

### **2. The `w` Command**

Provides a more detailed, real-time "dashboard" of user activity.

- **Header:** Shows the same info as the `uptime` command (Current time, uptime, number of users, and **load average**).
    
- **Key Columns:**
    
    - **IDLE:** How long since the user last typed a command.
        
    - **WHAT:** What specific command or process the user is currently running.
        
    - **JCPU/PCPU:** CPU time used by the user's processes.
        

## **Tracking History (`last`)**

While `who` and `w` show who is on the system **now**, the `last` command shows who was on the system **in the past**.

- **Source File:** It reads from `/var/log/wtmp`.
    
- **Function:** Displays a list of all logins, logouts, and system reboots.
    
- **Output:** Shows the username, terminal, source (IP/Host), date/time, and duration of the session.
    
- **Reboot Records:** Instead of a login location, these lines show the Linux kernel version used during that boot.
