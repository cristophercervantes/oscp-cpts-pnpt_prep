# Introduction to Linux Files
Command Line Importance: While some Linux distros have GUIs, mastering the command line for file and directory manipulation is essential.

Case Sensitivity: Linux is entirely case-sensitive (inherited from Unix). `hello.txt`, `HELLO.txt`, and `Hello.txt` are three distinct files.

Character Standards: Linux uses UTF-8, which is based on ASCII.

  Run `man -s 7 ascii` to learn more about the ASCII standard.
# Globbing (Wild Cards)

• Definition: Glob characters are symbols with special meanings interpreted by the shell.

• Purpose: They allow for pattern matching to affect multiple files at once (e.g., all files with a certain extension).

• Shell Interpretation: Globs are processed by the shell before the command is executed, making them compatible with almost any command.

• Common Usage: Often demonstrated using the `echo` command.

### **1. Core Globbing Characters (Wildcards)**

Globbing characters are interpreted by the shell to match patterns of filenames.

- **Asterisk (`*`)**: Represents **zero or more** of any character.
    
    - `t*`: Matches anything starting with 't' (e.g., `terminfo`, `timezone`).
        
    - `*.d`: Matches anything ending in '.d'.
        
    - `r*.conf`: Matches anything starting with 'r' and ending in '.conf'.
        
- **Question Mark (`?`)**: Represents **exactly one** character.
    
    - `t???????`: Matches a string starting with 't' followed by exactly 7 characters.
        
    - `*.???`: Matches any file with a exactly a three-letter extension.
        
- **Brackets (`[]`)**: Matches **one** single character from a specific set or range.
    
    - `[gu]*`: Matches anything starting with either 'g' or 'u'.
        
    - `[a-d]*`: Matches anything starting with a, b, c, or d.
        
    - `*[0-9]*`: Matches any filename containing at least one number.
        
    - **Note on Ranges**: Ranges are based on **ASCII** order. An invalid order (e.g., `[9-0]`) will result in no matches.
        
### **2. Shell Expansion Mechanism**

- **The Shell vs. The Command**: The shell expands the glob pattern **before** the command (like `ls` or `echo`) is actually executed.
    
- **Example**: If you type `echo /etc/a*`, the shell finds all matches and replaces `/etc/a*` with the actual list of files before `echo` runs.
    
### **3. The `ls` Globbing "Trap"**

The `ls` command behaves differently based on the type of argument it receives, which can cause confusion when globbing:

1. **If the argument is a File**: `ls` prints the filename.
    
2. **If the argument is a Directory**: `ls` prints the **contents** of that directory.
    

**The Problem:**

If a glob matches a directory name, `ls` will list the files _inside_ that directory rather than just the directory name itself.

- _Example_: `ls /etc/x*` might match the directory `/etc/xdg`. Instead of showing `/etc/xdg`, `ls` shows the files inside it (e.g., `systemd`, `autostart`), which can make the output look incorrect or confusing.
    

**The Solution:**

Always use the **`-d`** option with `ls` when using globs.

- **`ls -d`**: Forces `ls` to display the name of the directory itself, not its contents.
    
- _Corrected Example_: `ls -d /etc/x*` $\rightarrow$ Result: `/etc/xdg`

# Copy
The `cp` command is used to create copies of files and directories. It requires at least two arguments: a **source** (what you are copying) and a **destination** (where the copy goes).

### Command Structure & Basic Use

- **Syntax:** `cp [options] source destination`
    
- **Behavior:** If successful, `cp` provides no output ("no news is good news").
    
- **Destination Rules:**
    
    - If the destination is a **directory**, the file keeps its original name.
        
    - To **rename** the file during the copy, specify a new filename in the destination path (e.g., `cp file.txt ~/newname.txt`).
        

### Useful Options

|**Option**|**Name**|**Description**|
|---|---|---|
|**`-v`**|**Verbose**|Displays what is happening (e.g., `` `/etc/hosts' -> `/home/sysadmin/hosts' ``).|
|**`-i`**|**Interactive**|**Safeguard:** Prompts you for confirmation (`y/n`) before overwriting an existing file.|
|**`-n`**|**No Clobber**|**Safeguard:** Automatically answers "no" to overwriting. It will not copy if the file already exists.|
|**`-r`** or **`-R`**|**Recursive**|Required to copy **directories** and all their contents (files and subdirectories).|

### Important Safeguards & Behaviours

#### **The Danger of Overwriting**

By default, `cp` is **destructive**. If the destination filename already exists, `cp` will overwrite it immediately without warning, destroying the original data in that destination file.

> **Tip:** Use `cp -i` if you are unsure if a file already exists in the destination.

#### **Copying Directories**

Standard `cp` will result in an error if you try to copy a directory. You must use the **recursive** flag:

- `cp -r source_dir destination_dir`
    
- **Note:** While `-r` and `-R` both mean recursive for `cp`, be careful with other commands. For example, in `ls`, `-r` means "reverse sort," while `-R` means "recursive."

# **Moving & Renaming Files (`mv`)**

The `mv` command is used to move a file or directory from one location to another. Unlike `cp`, the `mv` command removes the file from its original source after placing it in the destination.

### ** Moving vs. Renaming**

The `mv` command serves two purposes depending on how the destination is specified:

- **Moving:** If the destination is a directory, the file is relocated there.
    
- **Renaming:** If the destination is a new filename (within the same directory or a different one), the file is renamed.
    
    - _Example:_ `mv oldname.txt newname.txt` "moves" the file to the same folder but with a new name.
        
    - _Example:_ `mv file.txt Videos/newname.txt` moves **and** renames the file simultaneously.
        

> **Key Difference:** To move a file, you need specific permissions to **remove** it from the source directory. If you lack these, you will see a `Permission denied` error.

### ** Move Options**

The `mv` command shares several safety and feedback options with the `cp` command:

|**Option**|**Name**|**Meaning**|
|---|---|---|
|**`-i`**|**Interactive**|Prompts for confirmation before overwriting an existing file at the destination.|
|**`-n`**|**No Clobber**|Prevents overwriting. If a file with the same name exists at the destination, the move won't happen.|
|**`-v`**|**Verbose**|Shows the move as it happens (e.g., `'file.txt' -> 'Videos/file.txt'`).|

### ** Moving Directories**

A major difference between `cp` and `mv` is how they handle directories:

- **No `-r` required:** The `mv` command moves directories by default. You do **not** need the recursive option to move an entire folder and its contents.

# **Creating Files (`touch`)**

The `touch` command is the simplest way to create a new file.

- **Function:** Creates a new, empty file if the file does not already exist.
    
- **Result:** A file with a size of **0 bytes**.
    
- **Use Case:** Useful as a "placeholder" or for services that require a specific file to exist to function correctly.
    
- **Example:** `touch sample.txt`
    

# ** Removing Files & Directories (`rm`, `rmdir`)**

The `rm` (remove) command is used to delete data. **Warning:** In the Linux CLI, there is no "Trash" or "Recycle Bin"—once a file is removed, it is gone.

### ** Deleting Directories**

- **The Default Rule:** `rm` alone will **not** delete a directory; it will return an error: `Is a directory`.
    
- **Recursive Delete (`rm -r`):** To delete a directory and everything inside it (files and subdirectories), use the `-r` option.
    
    - _Example:_ `rm -r Videos`
        
- **Empty Directories Only (`rmdir`):** You can use `rmdir` to delete a directory, but it **must be empty**. If there is even one file inside, it will fail.
    

> **Pro Tip:** Always use **`rm -i`** (interactive) when deleting directories. It will ask for confirmation before deleting, preventing accidental mass data loss.

# ** Creating Directories (`mkdir`)**

To create a new directory (folder), use the `mkdir` command.

- **Syntax:** `mkdir directory_name`
    
- **Example:** `mkdir test` creates a folder named "test" in your current location.
