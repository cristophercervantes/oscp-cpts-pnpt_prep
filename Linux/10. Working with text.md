# Introduction
## 1. Introduction to Text Files & Redirection

- **Text Files:** Contain only plain text (no special formatting like bold/italics). Most Linux configuration files are text-based.
    
- **Manipulation:** Linux provides numerous commands to view, modify, and combine these files.
    
- **Redirection:** Shell output can be redirected from the terminal window into other files or commands, making the CLI environment highly flexible.
    

## 2. Basic Viewing Commands

### The `cat` Command (Concatenate)

- **Primary Use:** Display the entire content of a file to the terminal (Standard Output).
    
- **Syntax:** `cat [filename]`
    
- **Capability:** Can also be used to create new files or combine multiple files into one.
    
- **Drawback:** Not ideal for large files because it "dumps" the whole content at once without pausing.
    

## 3. Pager Commands (`less` vs. `more`)

Pagers allow you to view large files one page at a time.

|**Feature**|**more**|**less**|
|---|---|---|
|**Availability**|Always available on all UNIX/Linux systems.|Default on most, but might not be on minimal installs.|
|**Navigation**|Primarily forward movement.|Advanced forward and backward movement.|
|**Search**|Basic.|Advanced (regex support).|

### Pager Navigation Keys (Work in both `more` and `less`)

- **Spacebar:** Move forward one window.
    
- **B:** Move backward one window.
    
- **Enter:** Move forward one line.
    
- **H:** Open Help screen.
    
- **Q:** Exit/Quit the pager.
    

### Searching within `less`

1. **Forward Search:** Press `/` followed by your search term (e.g., `/error`).
    
2. **Backward Search:** Press `?` followed by your term.
    
3. **Repeat Search:**
    
    - `n`: Move to the **next** match.
        
    - `N` (Shift+N): Move to the **previous** match.
        

## 4. Viewing File Fragments (`head` & `tail`)

These commands are used to "peek" at the beginning or end of a file. By default, both show **10 lines**.

### The `head` Command (Top of file)

- **Default:** `head filename` (shows first 10 lines).
    
- **Specific number:** `head -n 5 filename` or `head -5 filename`.
    
- **GNU Special Feature:** `head -n -3` shows the entire file **except** the last 3 lines.
    

### The `tail` Command (Bottom of file)

- **Default:** `tail filename` (shows last 10 lines).
    
- **Specific number:** `tail -n 5 filename`.
    
- **Start at line X:** `tail -n +25` shows everything from line 25 to the very end.
    
- **The Follow Option (`-f`):**
    
    - `tail -f /var/log/syslog`
        
    - **Use Case:** Critical for System Administrators to watch log files update in real-time while troubleshooting.

# Command Line Pipes
## 1. What is a Pipe?

The **pipe character `|`** is a powerful operator that connects the **Standard Output (stdout)** of one command to the **Standard Input (stdin)** of another.

- **Logic:** `Command A | Command B`
    
    - Command A runs first.
        
    - Instead of displaying its results on the screen, those results are "piped" into Command B.
        
    - Command B processes that data and then displays its own result.
        

## 2. Refining Data with Pipes

Pipes are most useful when dealing with commands that generate too much information to read at once (like `ls /etc`).

### Common Use Case: `ls` with `head` or `tail`

If a directory contains hundreds of files, you can limit the view:

- `ls /etc | head`: Shows only the first 10 files in the directory.
    
- `ls /etc | tail -n 5`: Shows only the last 5 files in the directory.
    

## 3. Multiple Pipes & Command Order

You can chain multiple commands together. However, **the order of commands matters** because each command only "sees" the data provided by the one immediately before it.

### Example: The Numbering Experiment (`nl` command)

The `nl` command adds line numbers to its input. Notice how changing the order of `nl` and `tail` changes the output:

#### Case A: `nl` before `tail`

> `ls /etc/ssh | nl | tail -5`

- **Step 1:** `ls` lists all files.
    
- **Step 2:** `nl` numbers all files (e.g., 1 to 10).
    
- **Step 3:** `tail -5` grabs the last 5 lines.
    
- **Result:** You see the original line numbers (e.g., **6, 7, 8, 9, 10**).
    

#### Case B: `tail` before `nl`

> `ls /etc/ssh | tail -5 | nl`

- **Step 1:** `ls` lists all files.
    
- **Step 2:** `tail -5` grabs only the last 5 files.
    
- **Step 3:** `nl` numbers the 5 files it received.
    
- **Result:** You see new line numbers starting from **1** (e.g., **1, 2, 3, 4, 5**).

# Input/Output Redirection
## 1. The Three Standard Streams

Every command in Linux uses three standard "channels" or streams to communicate.

|**Stream Name**|**Description**|**Channel ID**|**Default Device**|
|---|---|---|---|
|**STDIN**|Standard Input (data sent to the command)|0|Keyboard|
|**STDOUT**|Standard Output (successful command results)|1|Terminal Screen|
|**STDERR**|Standard Error (error messages)|2|Terminal Screen|

## 2. Redirecting Output (STDOUT)

You can send the results of a command to a file instead of the screen.

- **Overwrite (`>`):** Creates a new file or replaces the content of an existing file.
    
    - `echo "Hello" > file.txt`
        
- **Append (`>>`):** Adds the output to the end of an existing file without deleting what is already there.
    
    - `echo "World" >> file.txt`
        

## 3. Redirecting Errors (STDERR)

Since STDOUT and STDERR are different streams, you must tell Linux specifically if you want to redirect errors.

- **Redirect Error:** Use `2>` to send error messages to a file.
    
    - `ls /fake_dir 2> error.txt`
        
- **Why use this?** It keeps your screen clean when running scripts that might generate many "permission denied" or "file not found" messages.
    

## 4. Redirecting Multiple Streams

You can handle successful output and error messages simultaneously.

- **Separate Files:** Redirect each to its own destination.
    
    - `ls /etc /fake > success.txt 2> error.txt`
        
- **Same File (`&>`):** Combine both into one file.
    
    - `ls /etc /fake &> all_output.txt`
        
    - _Note: In the combined file, Linux usually groups all errors at the top and successful output below them._
        

## 5. Redirecting Input (STDIN)

This tells a command to get its data from a file instead of the keyboard.

- **The Operator:** `<`
    
- **The Use Case:** Most commands (like `cat` or `head`) accept filenames as arguments. However, some commands—like **`tr` (translate)**—do _not_ accept filenames.
    
- **Example with `tr`:**
    
    - `tr 'a-z' 'A-Z' < lower.txt` (Converts everything in `lower.txt` to uppercase).
        

## 6. Combining Everything

You can redirect input and output in a single command line:

> `tr 'a-z' 'A-Z' < input.txt > output.txt`

- **Step 1:** Read data from `input.txt`.
    
- **Step 2:** `tr` converts it to uppercase.
    
- **Step 3:** Save the result into `output.txt`.

# Sorting Files or Input
## 1. The `sort` Command

The `sort` command rearranges lines in a file. By default, it sorts alphabetically (dictionary order) starting from the very first character of each line.

- **Basic Syntax:** `sort [filename]`
    
- **Example:** If you sort the `/etc/passwd` file, it will move lines starting with 'a' to the top and 'z' to the bottom.
    

## 2. Fields and Delimiters

In many Linux files (like `.csv` or `/etc/passwd`), data is organized into columns called **fields**. These fields are separated by a **delimiter** (a specific character like a comma, colon, or space).

To sort by a specific column, you need to tell the `sort` command two things:

1. What the separator is (`-t`).
    
2. Which column to look at (`-k`).
    

## 3. Key `sort` Options

|**Option**|**Function**|**Description**|
|---|---|---|
|**`-t`**|**Field Delimiter**|Specifies the character that separates columns (e.g., `-t:` for colons, `-t,` for commas).|
|**`-k`**|**Field Number**|Specifies which "key" (column) to sort by (e.g., `-k3` for the 3rd column).|
|**`-n`**|**Numeric Sort**|Forces the command to treat numbers as values (e.g., so 10 comes after 2).|
|**`-r`**|**Reverse Sort**|Reverses the output (Z to A, or High to Low).|

## 4. Practical Examples

### Case A: Numeric Sort by UID

To sort the `/etc/passwd` file by the User ID (the 3rd field), which uses colons as separators:

> `sort -t: -k3 -n mypasswd`

- `-t:` : Uses the colon as the separator.
    
- `-k3` : Looks at the 3rd field.
    
- `-n` : Sorts numerically ($0, 1, 2...$) rather than alphabetically ($0, 1, 10, 2...$).
    

### Case B: Complex/Multi-Level Sorting

You can sort by multiple criteria at once by adding more `-k` options. The command handles them in the order they appear.

**Example Command:** `sort -t, -k2 -k1n -k3 os.csv`

1. **`-k2`**: First, sort by the Operating System (Column 2).
    
2. **`-k1n`**: If there's a tie in Column 2, sort by the Year (Column 1) numerically.
    
3. **`-k3`**: If there's still a tie, sort by the Last Name (Column 3).

# Viewing File Statistics
## 1. The `wc` Command (Word Count)

The `wc` command is used to count the number of lines, words, and bytes in a file. It is a simple but essential tool for verifying file sizes or the results of a command.

### Default Behavior

When you run `wc [filename]` without any options, it displays four columns in this specific order:

1. **Lines:** The number of newline characters.
    
2. **Words:** The number of strings of characters delimited by whitespace.
    
3. **Bytes:** The total size of the file in bytes (usually 1 character = 1 byte in plain text).
    
4. **Filename:** The name of the file being counted.
    

**Example:**

`wc /etc/passwd`

> `35 56 1710 /etc/passwd`

## 2. Specific Statistics Options

You can use options to display only the specific data points you need. These can be used individually or combined (e.g., `-lw`).

|**Option**|**Function**|**Description**|
|---|---|---|
|**`-l`**|**Lines**|Counts only the number of lines.|
|**`-w`**|**Words**|Counts only the number of words.|
|**`-c`**|**Bytes**|Counts only the total number of bytes (characters).|
|**`-m`**|**Characters**|Counts the number of characters (useful for multi-byte encodings).|

## 3. Counting Multiple Files

If you provide multiple filenames as arguments, `wc` will:

- Display the statistics for **each individual file**.
    
- Provide a **`total`** line at the bottom that sums up the statistics for all files.
    

## 4. Power of the Pipe: Counting Command Output

The most common "real world" use for `wc -l` is counting the number of items returned by another command.

- **Counting Files:** To see how many files are in a directory:
    
    `ls /etc | wc -l`
    
- **Counting Users:** To see how many user accounts are on the system:
    
    `cat /etc/passwd | wc -l`
    

# Filter File Sections
## 1. The `cut` Command

The `cut` command allows you to "snip" out specific columns or character ranges from a file. It is most commonly used to extract data from delimited files (like CSVs or system databases).

## 2. Extracting by Fields (Delimited Files)

When data is separated by a specific character (like a colon or comma), use the **field** options.

|**Option**|**Function**|**Description**|
|---|---|---|
|**`-d`**|**Delimiter**|Defines the character that separates fields. (Default is **Tab**).|
|**`-f`**|**Fields**|Specifies which columns to extract.|

### Field Selection Syntax:

- **Comma (`,`):** Select specific fields (e.g., `-f1,5` for fields 1 and 5).
    
- **Hyphen (`-`):** Select a range (e.g., `-f5-7` for fields 5, 6, and 7).
    
- **Combined:** `-f1,5-7` extracts the first field AND fields 5 through 7.
    

**Example:**

> `cut -d: -f1,5-7 mypasswd`

- This tells Linux: "Use the **colon** as a separator, and show me the **username** (1) and the **home directory info** (5-7)."
    

## 3. Extracting by Character Position

Sometimes data isn't separated by a delimiter but is instead "fixed-width" (where data always stays in the exact same spot on the line).

|**Option**|**Function**|**Description**|
|---|---|---|
|**`-c`**|**Characters**|Extracts data based on the character count from the start of the line.|

### Character Selection Examples:

- `cut -c1-10`: Extracts the first 10 characters of every line.
    
- `cut -c50-`: Extracts from the 50th character to the **end of the line**.
    

**Practical Use Case:**

When running `ls -l`, the file permissions always occupy the first 10–11 characters.

> `ls -l | cut -c1-11`

- This command filters the long list to show _only_ the permission strings (e.g., `drwxr-xr-x`).
    

## 4. `cut` vs. `sort` vs. `grep`

- **`sort`**: Changes the **order** of the lines.
    
- **`cut`**: Changes the **width** of the lines (removes unwanted columns).
    
- **`grep`**: Changes the **number** of lines (keeps only lines matching a pattern).

# Filter File Contents
## 1. The `grep` Command

The `grep` command (Global Regular Expression Print) searches for **patterns** within files or command output. If a line contains the pattern, `grep` prints the entire line to the screen.

- **Basic Syntax:** `grep [options] "pattern" [file]`
    
- **Example:** `grep bash /etc/passwd` (Finds every user account that uses the bash shell).
    

## 2. Essential `grep` Options

These flags change how `grep` searches or what it displays in the results.

|**Option**|**Function**|**Description**|
|---|---|---|
|**`--color`**|**Highlight**|Highlights the matching text in red (often on by default in modern Linux).|
|**`-i`**|**Ignore Case**|Makes the search case-insensitive (matches "The", "THE", and "the").|
|**`-v`**|**Invert Match**|Shows everything **except** the lines that match the pattern.|
|**`-c`**|**Count**|Instead of showing the lines, it tells you **how many** lines matched.|
|**`-n`**|**Line Number**|Shows the original line number from the file next to the result.|
|**`-w`**|**Whole Word**|Matches only if the pattern is a standalone word (won't find "are" inside "beware").|

## 3. Practical Use Cases

### Finding Information

If you want to find your user information in the system database:

> `grep "sysadmin" /etc/passwd`

### Cleaning Up Output (Invert Match)

If you want to see which users are **active** (not "nologin" users):

> `grep -v "nologin" /etc/passwd`

### Real-Time Troubleshooting

Combine `tail -f` with `grep` to watch a log file for a specific error as it happens:

> `tail -f /var/log/syslog | grep "error"`

## 4. Comparing `grep` and `grep -w`

The `-w` (Whole Word) option is crucial for accuracy.

- **Searching for "it":**
    
    - `grep "it"` will match: **it**, b**it**, ed**it**, and spir**it**s.
        
    - `grep -w "it"` will match: **it**.

# Basic Regular Expressions
## 1. What are Regular Expressions?

Regular Expressions (Regex) are a standardized set of symbols used to define search patterns.

- **Normal Characters:** Match themselves (e.g., `a` matches "a").
    
- **Special Characters (Metacharacters):** Have functional meanings (e.g., `.` matches any character).
    

Always wrap your regex in **single quotes** (e.g., `'^root'`) to prevent the Linux shell from misinterpreting the special characters before they reach the `grep` command.

## 2. Basic Regular Expressions (BRE)

|**Character**|**Matches...**|**Example**|**Result**|
|---|---|---|---|
|**`.`**|**Any single character** (except newline).|`'r..f'`|Matches `reef`, `roof`, `r#%f`|
|**`[ ]`**|**One character from a list/range**.|`'[0-9]'`|Matches any single digit|
|**`[^ ]`**|**One character NOT in the list**.|`'[^0-9]'`|Matches any non-numeric character|
|**`*`**|**Zero or more** of the _previous_ character.|`'re*d'`|Matches `rd`, `red`, `reeed`|
|**`^`**|**Beginning of the line** (Anchor).|`'^root'`|Matches "root" only at the start|
|**`$`**|**End of the line** (Anchor).|`'r$'`|Matches lines ending in "r"|
|**`\`**|**Escapes** a special character.|`'r\*'`|Searches for a literal asterisk `*`|

## 3. Deep Dive: The Brackets `[ ]`

The brackets act as a "pick one" tool.

- **Ranges:** You can use `-` to define a range based on the **ASCII table** (e.g., `[a-z]` or `[0-9]`).
    
- **Invalid Ranges:** `[z-a]` fails because "z" has a higher ASCII value than "a".
    
- **Negation:** `[^a-z]` finds lines containing at least one character that is **not** a lowercase letter.
    

## 4. Anchor Characters

Anchors don't match characters; they match **positions**.

- **`^` (Caret):** Must be the **first** character in your pattern.
    
- **`$` (Dollar):** Must be the **last** character in your pattern.
    
- **Combined:** `^hello$` only matches a line that says "hello" and nothing else.
    

## 5. Extended Regular Expressions (ERE)

To use these, you must use **`grep -E`** (or the older `egrep` command).

|**Character**|**Meaning**|**Example**|**Result**|
|---|---|---|---|
|**`?`**|Previous character is **optional** (0 or 1).|`'colou?r'`|Matches `color` and `colour`|
|**`+`**|Previous character **at least once** (1 or more).|`'e+'`|Matches `red` but **not** `rd`|
|**`|`**|**Logical OR** (Alternation).|`'gray|



Confusing points:

- **`a*`**: matches (nothing), `a`, `aa`, `aaa`...
    
- **`a+`**: matches `a`, `aa`, `aaa`... (must have at least one)
    
- **`a?`**: matches (nothing) or `a` (only zero or one)
