# Introduction
## 1. What is a Shell Script?

A **shell script** is essentially a text file containing a list of commands. Instead of typing them one by one, you run the file, and the shell executes each line in order.

### Why Script?

- **Automation:** Turn complex, multi-step tasks into a single command.
    
- **Consistency:** Ensure the same steps are followed every time without human error.
    
- **Logic:** Scripts can "think" by testing for files or checking if a previous command succeeded.
    

## 2. Anatomy of a Script: The Shebang

Most scripts begin with a special first line called the **Shebang** (`#!`). This tells the system which interpreter (shell) to use to run the code.

- `#!/bin/bash`: Uses the Bourne Again Shell (standard for Linux).
    
- `#!/bin/sh`: Uses the basic system shell.
    
- `#!/usr/bin/python3`: Uses the Python interpreter.
    

## 3. Creating and Running Scripts

To make a script work, you generally follow these three steps:

### Step 1: Create the file

Use a text editor to write your commands and save them (e.g., `myscript.sh`).

### Step 2: Make it Executable

By default, new text files don't have "permission" to run as programs. You must change the file's mode:

> `chmod +x myscript.sh`

### Step 3: Run it

Since the current directory is usually not in your system's search path (`$PATH`), you must specify the location:

> `./myscript.sh` (The `./` means "look in the current folder").

## 4. The Nano Text Editor

While advanced users often use `vi` or `vim`, **GNU Nano** is the most beginner-friendly editor. It is "modeless," meaning you just start typing.

### Essential Nano Shortcuts

The `^` symbol in Nano represents the **Control** key.

|**Shortcut**|**Action**|**Description**|
|---|---|---|
|**`Ctrl + O`**|**WriteOut**|Saves the file without exiting.|
|**`Ctrl + X`**|**Exit**|Closes Nano (it will ask to save if you have changes).|
|**`Ctrl + K`**|**Cut**|Deletes/Cuts the current line.|
|**`Ctrl + U`**|**Uncut**|Pastes the line you just cut.|
|**`Ctrl + W`**|**Where Is**|Opens a search tool to find text.|

# Scripting Basics
## 1. Variables: Storing Information

Variables act as temporary containers for data.

- **Assignment:** `NAME="Value"` (Note: **No spaces** around the `=` sign).
    
- **Accessing:** Use a `$` before the name to use the value (Interpolation).
    
- **Command Substitution:** Use backticks `` `command` `` to save the output of a command into a variable.
    

### Special Variables

- **`$0`**: The name of the script.
    
- **`$1` - `$9`**: Arguments passed to the script (e.g., `./run.sh argument1`).
    
- **`$?`**: The **exit code** of the last command (0 = success, 1+ = error/failure).
    

## 2. Conditionals: Making Decisions

Conditionals allow your script to "branch" or choose a path based on a test.

### The `if` Statement

The `if` statement checks the exit code of a command. If it's `0`, the code runs.

Bash

```
if grep -q "root" /etc/passwd; then
  echo "User exists"
else
  echo "User not found"
fi
```

### The `test` Command and `[ ]`

To compare strings or numbers, we use `test` (often written as `[`).

- **File Tests:** `[ -f file ]` (is it a file?), `[ -d dir ]` (is it a directory?).
    
- **String Tests:** `[ "$A" = "$B" ]` (equal?), `[ -z "$A" ]` (is it empty?).
    
- **Numeric Tests:** `[ $A -eq $B ]` (equal?), `[ $A -gt $B ]` (greater than).
    

### The `case` Statement

When you have many possible matches for one variable, `case` is cleaner than multiple `elif` statements.

Bash

```
case "$1" in
  start) echo "Starting..." ;;
  stop)  echo "Stopping..." ;;
  *)     echo "Usage: start|stop" ;;
esac
```

## 3. Loops: Repeating Tasks

Loops execute the same block of code multiple times.

### The `for` Loop

Used when you have a **fixed list** of items (files, names, servers).

- `for FILE in *; do ... done`: Runs the code for every file in the current folder.
    
- `for NAME in Alice Bob; do ... done`: Runs specifically for Alice and then Bob.
    

### The `while` Loop

Used when you want to repeat an action **as long as a condition is true**.

Bash

```
i=0
while [ $i -lt 5 ]; do
  echo "Number: $i"
  i=$(( $i + 1 )) # Increment the counter
done
```
